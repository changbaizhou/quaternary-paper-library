# 论文知识工作区设计

## 目标与范围

本轮只完善七项：参考文献解析、论文关系图、图表目录、文件夹批量导入、第四纪术语词典、引用写作工作台和证据对比。继续保持单机、SQLite、本地优先；不增加阅读任务看板、本地大模型、后台目录监听或新的 AI 调用。

## 数据与解析

SQLite migration 8 新增五组数据：`paper_references` 保存参考文献原文、页码、DOI、题名、年份和已入库匹配；`paper_relations` 保存确认的引用、支持、反对、相关和自定义关系；`paper_assets` 保存图题、表题和页码；`custom_terms` 保存用户术语、别名、类别和释义；`writing_drafts` 保存项目写作草稿、引用样式和已引用论文。

参考文献和图表只从已建立的页级 PDF/OCR 文本中解析。参考文献优先按 DOI 匹配，缺少 DOI 时才用规范化题名精确匹配；不猜测未入库论文。图表目录识别 `Figure`、`Fig.`、`Table`、`图`和`表`题注，只提供标题和页码跳转，不裁取图片。

同区域、同方法和高相似论文采用现有结构化分类与元数据的可解释评分，按需计算，不把所有组合写入数据库。人工关系和确认的引用关系持久化。

## 导入与术语

文件夹导入使用浏览器目录选择器，把目录中的 PDF 作为一次批量上传发送到现有上传接口。每篇仍进入待确认流程，继续使用现有哈希去重、OCR、元数据识别和确认入库规则。系统不常驻监控用户文件夹。

内置第四纪分类词表继续来自 `taxonomy.js`。用户可维护自定义术语的规范名、别名、类别和释义；自定义别名与内置词表一起参与语义扩展，不需要重新索引论文。

## 界面

上传区增加“导入文件夹”。论文详情增加“论文知识”命令，打开大尺寸对话框，集中展示关系图、参考文献和图表目录；所有论文节点、参考条目和图表条目均可跳回论文或 PDF 页。

页头增加“术语”命令，打开可检索、增加、编辑和删除自定义术语的对话框。关系图使用本地 SVG，以当前论文为中心，最多显示 20 个直接关系节点，避免引入图形库。

项目证据区增加立场和证据类型筛选、支持/反对/混合统计、项目关系行内编辑，以及项目写作草稿。写作草稿可从证据行插入摘录和文中引用，自动生成当前已引用论文的参考文献预览；草稿正文和引用论文 ID 保存到 SQLite。

## 错误与安全

解析失败不影响论文和页级索引。派生索引可以重复重建且结果稳定。所有 ID、类型、文本长度和版本号由服务端验证；接口不返回绝对路径、API Key 或整篇正文。批量导入仍只接受 PDF，术语和草稿按纯文本处理。

## 验收

1. 旧论文和新论文都能重建参考文献与图表目录。
2. 已入库 DOI 引用自动建立关系，未匹配条目保留原文但不建立关系。
3. 关系图节点可打开论文，图表条目可打开对应页。
4. 选择文件夹后只上传其中 PDF，并进入现有待确认列表。
5. 自定义术语保存后立即影响语义检索，关闭语义扩展后不生效。
6. 写作草稿刷新后仍存在，插入证据会同步引用和参考文献预览。
7. 证据筛选、统计、行内编辑和原有导出同时可用。
