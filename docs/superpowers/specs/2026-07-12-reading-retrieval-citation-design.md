# 论文阅读、检索与引用工作台设计

## 目标

把第四纪论文库从“文件和元数据管理器”升级为完整研究工作台，使用户能够：

1. 搜索题名、作者、摘要、全文和笔记，并直接跳到命中的 PDF 页。
2. 在阅读器中高亮、翻译、批注和保存带页码的研究卡片。
3. 按研究项目组织论文、证据和争议观点，而不复制原文件。
4. 生成经过校验的 GB/T 7714、APA、BibTeX、RIS 和 CSL-JSON 引用。
5. 对论文库提问时只基于检索到的页级原文回答，并附论文与页码。

系统继续保持单机、单用户、本地优先。PDF、数据库、笔记和密钥不进入 Git；只有用户主动使用联网翻译或 AI 问答时，才发送查询和必要的原文片段。

## 非目标

- 不自动撰写论文或替用户形成最终学术结论。
- 不在本阶段构建复杂知识图谱。
- 不依赖云端向量数据库或外部文献托管服务。
- 不自动改写人工笔记、元数据或引用信息。
- 不自动合并含义相近但来源不同的摘录。

## 总体架构

核心数据单位从“整篇论文文本”扩展为“页级文本”。上传、OCR 和旧文献重处理都生成统一的 `paper_pages` 记录。全文检索、页码跳转、摘录、引用定位和 AI 检索共用这些记录，避免维护多套相互漂移的文本。

实现分为六个可独立验收的阶段：

1. 页级文本与 SQLite FTS5 全文检索。
2. PDF 页码跳转、高亮、摘录和研究卡片。
3. 引用校验、引用键和多格式导出。
4. 研究项目、阅读队列和证据对比。
5. 带页码证据的 AI 问答。
6. 真实资料库迁移、浏览器验收和文档。

每个阶段使用版本化 SQLite 迁移，迁移前自动创建数据库备份。任何阶段失败都不能破坏现有论文、原文件、笔记、阅读进度或书签。

## 一、页级全文与检索

### 数据模型

新增 `paper_pages`：

- `id`
- `paper_id`
- `page_number`
- `text`
- `text_source`：`pdf`、`ocr` 或 `mixed`
- `language`
- `character_count`
- `created_at`、`updated_at`

`paper_id + page_number` 唯一。一篇论文重处理时，在事务内替换全部页记录；提取失败则保留旧页记录。

新增 FTS5 虚拟表 `paper_pages_fts`，索引页文本，并通过触发器与 `paper_pages` 同步。论文题名、作者、摘要和笔记继续进入现有 `papers.search_text`，统一搜索 API 会合并元数据结果与页级结果。

### 文本提取

- 文本型 PDF 使用 PDF.js 按页提取，不再只保留整篇拼接文本。
- 页面文本少于阈值时，仅对该页执行 OCR。
- OCR 与 PDF 文本都存在时，保留字符更完整的一份，并记录来源。
- 默认处理全文；前台显示进度。过大文件可后台继续，论文先进入“索引中”状态。
- 旧论文通过批量索引命令处理，命令先备份数据库，支持中断后继续。

### 检索行为

新增 `GET /api/search`，参数包括：

- `q`
- `scope=all|metadata|fulltext|notes`
- 现有主题、区域、时期、材料、方法、指标筛选
- 项目、阅读状态、年份范围筛选
- 分页参数

结果包含论文 ID、题名、作者、命中范围、页码、带高亮标记的短摘要和相关度。点击全文结果调用现有阅读器并滚动到目标页。

中英文跨语言检索先使用本地第四纪术语表扩展，例如“黄土”扩展 `loess`，“古土壤”扩展 `palaeosol`。启用 Qwen 时可只发送用户查询生成额外英文检索词，不发送论文内容。首版不引入向量数据库。

## 二、标注、摘录与研究卡片

### 数据模型

新增 `annotations`：

- `id`、`paper_id`、`page_number`
- `kind`：`highlight`、`note`、`quote`
- `quote_text`
- `translated_text`
- `comment`
- `color`
- `text_selector_json`
- `version`
- `created_at`、`updated_at`

文本选择器保存页码、原文、前后文和页内字符位置，不依赖屏幕像素。PDF 缩放或窗口变化后仍可恢复高亮；无法精确定位时显示为“待重新定位”，不丢弃记录。

新增 `research_cards`：

- `id`、`annotation_id`、`paper_id`
- `summary`
- `personal_interpretation`
- `themes_json`
- `evidence_type`：`supports`、`opposes`、`method`、`background`、`uncertain`
- `created_at`、`updated_at`

卡片始终保留论文、页码和原始摘录。点击卡片返回原文页；删除卡片不删除原高亮，除非用户明确选择同时删除。

### 阅读器交互

选中文字后出现紧凑工具条：高亮、批注、翻译、保存摘录。工具条不遮挡所选文本，移动端显示在底部操作区。侧栏新增“本篇标注”，支持按颜色、类型和关键词筛选。

## 三、引用管理

### 元数据与校验

论文新增：

- `citation_key`
- `citation_status`：`unverified`、`verified`、`incomplete`
- `citation_checked_at`

引用键默认按 `第一作者姓 + 年份 + 题名首个有效词` 生成，并在库内保持唯一。用户修改作者、年份或题名后，已使用的引用键不自动变化，只提示可重新生成。

引用详情显示题名、作者、年份、期刊、卷、期、页码、DOI 和类型。缺失必要字段时标为“不完整”，正式导出前提示，但允许用户继续。

### 输出

继续支持 BibTeX，并新增：

- RIS
- CSL-JSON
- GB/T 7714—2015
- APA 7
- 文中引用：`作者（年份）`、`(Author et al., Year)`

支持单篇复制、勾选多篇批量复制和按研究项目导出。格式化逻辑放在独立引用模块中，并使用结构化元数据，不通过拼接现有显示文本实现。

## 四、研究项目与证据对比

### 数据模型

新增 `research_projects`：名称、描述、状态、创建和更新时间。

新增 `project_papers`：

- `project_id`、`paper_id`
- `priority`：1 至 5
- `stance`：`supports`、`opposes`、`mixed`、`background`、`unknown`
- `project_status`：`queued`、`reading`、`reviewed`
- `project_note`

论文文件仍只保存一份。一篇论文可以属于多个项目，每个项目拥有独立的立场、优先级和项目笔记。

### 工作台

新增“项目”视图，默认不是营销首页。项目内提供阅读队列、核心论文、待阅读、已阅读和证据表。证据表按论文显示年代范围、研究区、材料、方法、主要结论、立场和对应研究卡片，支持 CSV/Markdown 导出。

## 五、带出处的 AI 问答

### 检索与回答

AI 问答只在页级索引完成后启用。流程：

1. 使用 FTS5、筛选条件和项目范围检索候选页。
2. 选取不超过配置上限的原文片段。
3. 向已配置的 Qwen 接口发送问题、片段、论文 ID、题名和页码。
4. 要求返回结构化答案和引用编号。
5. 服务端校验每个引用编号必须来自已发送片段。

回答中的每个关键结论都显示可点击出处，格式为“论文题名，第 12 页”。点击后进入阅读器对应页面。模型未获得足够证据时必须回答“当前资料库证据不足”，不能补造引用。

### 保存与隐私

新增 `research_answers`，保存问题、答案、引用 JSON、项目范围、提供商和模型，不保存 API Key。界面明确标记“AI 生成”，并允许将某一条带出处结论转为研究卡片；不会自动写入人工笔记。

## 六、错误处理与数据安全

- 页级索引失败：论文仍可阅读，显示失败原因和“重新索引”。
- OCR 失败：保留 PDF 文本和旧索引，不用空结果覆盖。
- 标注冲突：使用版本号返回 409，保留本地编辑并提示重新加载。
- 引用导出缺字段：显示具体缺失字段，不产生看似完整的伪数据。
- AI 超时或接口失败：不影响检索和阅读；问题可重试。
- 所有批量索引、引用批量修复和项目批量操作前创建数据库备份。
- 完整备份包含 PDF、页级文本、标注、卡片、项目和 AI 回答；不包含 `local.env.bat`。

## API 边界

新增或扩展以下 API：

- `GET /api/search`
- `POST /api/papers/:id/reindex`
- `GET|POST /api/papers/:id/annotations`
- `PATCH|DELETE /api/annotations/:id`
- `GET|POST /api/research-cards`
- `PATCH|DELETE /api/research-cards/:id`
- `GET|POST /api/projects`
- `GET|PATCH|DELETE /api/projects/:id`
- `POST|PATCH|DELETE /api/projects/:id/papers`
- `GET /api/citations/export`
- `PATCH /api/papers/:id/citation`
- `POST /api/research/ask`
- `GET /api/research/answers`

所有修改操作校验 ID、状态和版本；API 响应不返回本地绝对路径或密钥。

## 前端结构

顶层导航扩展为：论文库、项目、回收站、维护。搜索框保留在论文库主工作区，搜索结果替换中央列表；点击全文命中后进入原阅读器。阅读器增加标注工具条和右侧标注栏。项目视图复用中央工作区，不新增落地页或说明型首页。

论文详情操作区增加醒目的“移入回收站”按钮。点击后使用原生对话框显示论文题名并要求第二次明确确认，确认后调用现有软删除 API，刷新论文列表和回收站数量，并自动选择下一篇论文。该按钮不直接删除 PDF；彻底删除和清空操作仍只存在于回收站视图。

桌面 1366×768 保持三栏阅读效率；移动 390×844 采用单列顺序，操作条换行且无横向滚动。危险删除和批量覆盖继续使用二次确认。

## 测试与验收

### 自动测试

- 数据迁移、回滚、幂等和真实旧库副本演练。
- FTS5 中文、英文、短语、筛选和页码结果。
- OCR/PDF 混合页提取及失败时保留旧索引。
- 标注版本冲突、文本选择器恢复和卡片跳转。
- 引用键唯一性及 BibTeX、RIS、CSL-JSON、GB/T 7714、APA 输出。
- 项目多对多关系、项目删除不删除论文。
- AI 引用校验，拒绝不存在的页码或片段编号。
- API 路径、权限边界和绝对路径泄漏检查。

### 浏览器验收

使用临时数据库和生成的多页 PDF，验证：上传、分页索引、全文搜索、跳页、高亮、卡片、引用复制、项目证据表和带页码 AI 模拟回答。分别在 1366×768 和 390×844 截图，检查所有可见控件在视口内，PDF 画布非空且高亮与目标页一致。

### 真实资料库升级

停止本地服务，创建完整备份，在副本上迁移并对比论文、草稿、文件、笔记、书签和阅读进度。副本通过后才迁移真实库。升级后重新启动服务，验证现有 4 篇论文仍可打开，再执行可中断的全文索引。

## 实施顺序与完成标准

1. **全文检索完成**：任意命中结果在一次点击后打开正确论文和页码。
   同时补齐论文详情“移入回收站”入口，确认后论文从正常列表消失并可在回收站恢复。
2. **标注卡片完成**：刷新页面后高亮、摘录、翻译和个人理解均可恢复。
3. **引用完成**：同一批论文可生成结构正确的五种引用格式，缺字段明确提示。
4. **项目完成**：论文可加入多个项目，项目证据表可导出且不复制 PDF。
5. **AI 完成**：每个回答出处可点击，伪造引用被服务端拒绝。
6. **发布完成**：全量测试、浏览器测试、真实库副本迁移、凭据扫描和 Git 作者检查全部通过。
